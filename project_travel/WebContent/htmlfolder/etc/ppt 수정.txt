  데이터들을 뽑아내서 검색 결과페이지에서 검색된 결과를 보여줍니다.
  
7번 메인화면 카테고리 페이지

- 메인 페이지에서 카테고리를 구분하여 상품을 노출시키는 부분으로 상품 위에 마우스 포인터를 올려둘 경우
  상품에 이미지를 가르키는걸 알려주는 스타일 변화와 상품에 나라를 표현해주는 
텍스트가 출력되어 사용자가
  구분 가능할 수 있게 해줍니다.
  또한, slick lib 를 사용하여 상품을 버튼을 클릭하거나 마우스클릭한상태에서 
  끌어당기는 방식으로 여러 상품을 볼수 있게 적용

8번 상품 리스트 페이지

-상단에 네비바를 이용하여 상품 리스트 페이지로 들어오면 필터와 리스트가 출력되는데
 왼쪽 필터를 통해 js ajax를 사용해서 url 주소값으로 요청하여 서블릿으로 선택한 값을 가지고 찾아가 
 쿼리문을 요청하여 DB에서 데이터를 가져와 처리하여 jsp페이지에서 가져온데이터를 가지고 출력해준다.





9번 상품디테일 페이지

-상품 디테일에 상단 부분에 해당하는 구역으로 해당 상품에 데이터를 상품번호 컬럼을 이용해서 데이터들을
 가져와서 출력해줍니다. 오른쪽에는 상품에 가격 계산과 구글 api를 적용한 지도가 표시되어 있습니다.
 가격 계산 부분은 + 버튼을 누르게 될 경우 스크립트가 실행되어 현재에 인원 값과 가격값에 더 해주는 방식
 되어 값을 계산한다.
 새로운 변수에 위도와 경도 값을 넣어서 구글 맵 api 예제 코드를 적용하여 여행위치와 마커를 표시해주었다.
 
10번 상품디테일 페이지

-상품 디테일 하단 부분에 보여지는 부분으로 탭을 눌렀을 때 각각의 맞는 탭에 내용을 보여준다.
 위에 보이는 탭 내용은 해당 상품에 여행 내용을 설명하기위해서 새로 테이블을 만들어
 이 탭에서만 사용 할수 있게 만들었다 

11번 질문게시판 페이지

- 전형적인 CRUD 형식에 게시판 jsp 페이지에서 처리하여 DB에 요청하여 
insert, select , update, delete 를 처리한다.
수정과 삭제를 아무나 할수 없도록 현재 사용자가 다른 사람의 글에 들어갔을 경우에
수정 버튼과 삭제 버튼이 비활성화 되도록 조건문을 적용하여 안보이도록 조치.

12번 회원가입 페이지

- 아이디를 입력하여 중복확인 버튼을 클릭시 ajax 를 이용하여 url 주소값으로 요청하여 입력한 아이디값을
  DB에 요청하여 DB에 있는 아이디 값과 입력한 값을 비교하여 중복되었다면 변수에 값을 넣어줘서
  되돌아와서 변수 값에 따라서 알맞은 값을 아이디 텍스트 밑에 표시해준다.
